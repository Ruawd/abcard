# 兑换码系统使用文档

## 概述

兑换码系统用于控制 Web UI 的使用权限。用户必须输入有效的兑换码才能使用自动绑卡服务。

**核心规则：失败不扣次数**  
只有执行成功时才消耗一次额度，失败（支付被拒、超时等）或用户手动终止均不扣次数。

---

## 管理员操作

### 生成兑换码

使用 `admin_cli.py` 在服务器上生成兑换码：

```bash
# 生成 10 个一次性兑换码
python3 admin_cli.py generate 10

# 生成 5 个可用 3 次的兑换码
python3 admin_cli.py generate 5 --uses 3

# 生成 VIP 兑换码 (100次, 30天有效期)
python3 admin_cli.py generate 1 --uses 100 --expires 30 --note "VIP客户"

# 批量生成不同类型
python3 admin_cli.py generate 20 --uses 1 --note "试用"
python3 admin_cli.py generate 5 --uses 10 --note "标准"
python3 admin_cli.py generate 2 --uses 50 --expires 90 --note "高级"
```

### 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 第一个参数 | 生成数量 | 1 |
| `--uses N` | 每个码可用次数 | 1 |
| `--expires N` | 有效天数 (从生成时起) | 永不过期 |
| `--note TEXT` | 备注 | 空 |

### 查看兑换码

```bash
# 列出所有兑换码及使用情况
python3 admin_cli.py list
```

输出示例：
```
兑换码                  使用/总量   成功    失败 备注
----------------------------------------------------------------------
KED5-MBRS-NA79       2/     100      2      3 VIP客户
TFX6-69G5-GQFP       5/      10      5      1 标准
5BXF-QCMF-WZEG       1/       1      1      0 试用
```

### 查看单个兑换码详情

```bash
python3 admin_cli.py info KED5-MBRS-NA79
```

### 查看执行历史

```bash
python3 admin_cli.py history KED5-MBRS-NA79
```

输出示例：
```
[success] 2026-03-15T14:32:01 | user@example.com |
[failed]  2026-03-15T14:28:55 | user2@example.com | 支付被拒
[failed]  2026-03-15T14:25:10 | - | 用户手动终止
```

## 用户操作流程

### 1. 打开 Web UI

浏览器访问部署地址（如 `https://abc.example.com`）。

### 2. 输入兑换码

页面会显示兑换码输入框。输入管理员提供的兑换码，点击「验证」。

- ✅ 有效：进入操作界面，顶部显示「剩余 X/Y 次」
- ❌ 无效：显示错误原因（不存在 / 已用完 / 已过期）

### 3. 正常使用

验证通过后即可正常操作：
- 选择计划（Business / Plus）
- 填写卡片信息
- 点击「开始执行」

### 4. 额度扣减规则

点击「开始执行」时**立即扣减 1 次**，执行中 UI 显示减少后的剩余次数。

| 执行结果 | 额度处理 | 说明 |
|---------|----------|------|
| ✅ 成功 | **保持扣减** | 账号注册+绑卡成功 |
| ❌ 支付被拒 | **自动回退 +1** | 卡被拒绝，可更换卡后重试 |
| ❌ 超时 | **自动回退 +1** | 网络/验证码超时 |
| ❌ 注册失败 | **自动回退 +1** | 注册环节出错 |
| ⏹ 手动终止 | **自动回退 +1** | 用户主动停止 |

### 5. 查看历史

「历史」Tab 显示当前兑换码的所有执行记录，包含：
- 状态（成功/失败/运行中）
- 注册的邮箱
- 错误信息
- 执行时间

「账号」Tab 只显示成功的账号，可查看凭证详情。

### 6. 退出

点击页面顶部的「退出」按钮可切换到其他兑换码。

## 数据存储

| 文件 | 说明 |
|------|------|
| `data.db` | SQLite 数据库，包含兑换码和执行记录 |
| `test_outputs/` | 凭证文件（JSON），UI 运行时自动生成 |
| `logs/` | 执行日志 |

所有敏感数据（data.db, config.json, test_outputs/, logs/）已在 `.gitignore` 中排除。

---

## 数据库表结构

### codes 表（兑换码）

| 字段 | 类型 | 说明 |
|------|------|------|
| code | TEXT PK | 兑换码 (如 GMNF-2NB6-A98Y) |
| total_uses | INTEGER | 总可用次数 |
| used_count | INTEGER | 已使用次数（仅成功时 +1） |
| created_at | TEXT | 创建时间 |
| expires_at | TEXT | 过期时间（NULL = 永不过期） |
| note | TEXT | 管理员备注 |

### executions 表（执行记录）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增 ID |
| code | TEXT FK | 关联的兑换码 |
| email | TEXT | 注册的邮箱 |
| plan_type | TEXT | 计划类型 (team/plus) |
| status | TEXT | pending / running / success / failed |
| error_msg | TEXT | 错误信息 |
| result_json | TEXT | 完整结果 JSON |
| created_at | TEXT | 创建时间 |
| updated_at | TEXT | 最后更新时间 |

## 常见问题

**Q: 不同用户之间能看到彼此的账号吗？**  
A: 不能。账号数据按兑换码隔离：
| 场景 | 是否可见 |
|------|---------|
| 用户 A（码1）→ 用户 B（码2）的账号 | ❌ 不可见 |
| 用户 A（码1）→ 自己之前注册的账号 | ✅ 可见 |
| 两人共用同一个码 → 彼此的账号 | ✅ 可见（同码共享数据） |

**Q: 兑换码格式是什么？**  
A: `XXXX-XXXX-XXXX` 格式，大写字母 + 数字（排除了 O/0/I/1/L 等易混淆字符）。

**Q: 同一个兑换码可以多人同时使用吗？**  
A: 可以。每次执行独立计次，多人可以同时输入同一个码使用，只要剩余次数足够。

**Q: 支付失败后可以立即重试吗？**  
A: 可以。失败不扣次数，可以更换卡或调整设置后直接重试。

**Q: 如何备份数据？**  
A: 备份 `data.db` 文件即可，它包含所有兑换码和执行记录。

**Q: 如何重置兑换码次数？**  
A: 当前需手动操作数据库：
```bash
sqlite3 data.db "UPDATE codes SET used_count = 0 WHERE code = 'XXXX-XXXX-XXXX'"
```
